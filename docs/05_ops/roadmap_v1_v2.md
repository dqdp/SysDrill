# Roadmap v1 -> v2

## Phase 0 — Spec and content kernel
Deliverables:
- documentation bundle
- initial schemas
- initial concept/scenario pack
- first ADR set

## Phase 1 — Text Study Mode
Deliverables:
- content store
- 50–100 high-quality cards across 8–10 core topics
- study cards
- hint ladder support
- append-only interaction logging

## Phase 2 — Practice Mode + Evaluation
Deliverables:
- guided practice loop
- card families: recall / comparison / failure / sequencing / red-flag
- mini-scenarios
- rubric engine v1
- review report v1
- basic learner profile projection

## Phase 3 — Recommendation Engine v1
Deliverables:
- review queue
- next-best-action rules (`mode + intent + target`)
- rationale logging
- weak-area targeting
- spaced-review and remediation planning

## Phase 4 — Mock Interview
Deliverables:
- 3–5 mini-mock / mock scenarios for launch
- stricter runtime mode
- follow-up pressure logic
- mock readiness gating
- full review summary

## Phase 5 — Product hardening
Deliverables:
- analytics dashboards
- content versioning discipline
- recommendation tuning
- quality and assurance reports

## Phase 6 — v2 exploration
Possible additions:
- voice layer
- adaptive pacing
- advanced difficulty calibration
- B2B cohort analytics
- diagram-supported practice


## Authoring workflow posture

### v1
- bundled topic packages are acceptable
- canonical content and learning-design bindings remain schema-distinct
- template-assisted derivation is preferred over exhaustive handcrafted coverage
- launch focuses on a narrow, high-quality core rather than full ontology saturation

### Later
- if content scale or team size grows, the project may split bundled authoring into stricter editorial workflows with separate files, stronger review gates, and deeper provenance controls
